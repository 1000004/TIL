### 컴퓨터의 원리
* 컴퓨터 : **명령**을 **하나**씩 **순서**대로 **수행**하는 **기계**
* 프로그램 : 명령을 어떻게 수행할것인지 기술
* 컴퓨터는 단순한 기계 **1,0**을 가지고 사칙연산을 수행(3GHz 1초당 3G만큼 수행한다)
* 그림, 음악, 문자를 컴퓨터에 입력 가능하도록 숫자로 표현 숫자를 다시 0,1로 바꿀 수 있다.
 * **문자**
   * ASCII(아스키 코드)  0-255 1byte로 문자 표현(영문자까지 표현)
   * UNICODE 2byte (1-3byte) 모든 언어 표현가능
 * **그림** 
   * 화면이 1080x720이라고 볼때 가로 픽셀이 1080개 세로 픽셀이 720개 각 픽셀의 색깔의 나열
   * RGB R:255 G:255B:255 흰색이고 R:0G:0B:0은 검은색 각각 1byte로 표현 가능 A: 투명도 총 4byte로 색표현
 * **소리**
   * 파장과 진폭을 숫자로 표현
#### 컴퓨터의 동작 원리
* 프로그램은 요리에서 지침서
* **HDD** CPU가 사칙연산에 필요한 재료를 **HDD**에서 시리얼 버스을 통해 가져와 **MEMORY**에 올린다
* **MEMORY** CPU입장에서 메모리도 멀기 때문에 메모리에서 연산에 필요한 덩어리(Chunk)를 **CACHE**에 올려 놓는다
* (Chunk에는 연산에 필요한 데이터 이외에도 포함)(CACHE 크기 4K 4kbyte)
* **CACHE** 실제 연산에 들어가기 전에 준비를 해놓은 상태 **Register**에는 CPU가 A와 B를 ADD연산전 A,B가 올라가져 있다
* **Register**  **CPU**는 Register만보고 연산 수행
* 프로그램 읽기 수행(HDD에 있는것을 MEMORY에 올리는 것)은 시간이 많이 소모된다
#### 프로그래밍 언어란?
* 프로그램이 어떤 언어로 되어있는가 - 프로그래밍 언어
* 컴퓨터에게 사용자가 프로그램(명령과 순서를 쓴 문서)를 실행하라고 하면 기계어로 변경이 필요
* 컴퓨터에서 0,1 밖에 읽기 못한다(read)
* 자연어 처리 : 인간의 글를 문자 그대로 컴퓨터에서 읽는 것이 어렵다, 인간의 언어구조를 컴퓨터가 이해하는 것은 어려운 일
* 문서를 변환하는 과정이 필요 프로그램(글로 된 문서)을 컴퓨터가 바로 읽는 것은 불가능
* 켬퓨터가 읽을 수 있는 형태로 변경된 문서도 프로그램이고 인간의 글로 만들어진 문서도 프로그램
* 컴퓨터가 이해할 수 있게 변환 -> 기계어로 만든다
* 명령어에다가 숫자를 대입한 코드를 OPCODE라 부른다 ex ADD 3,4 (어셈블리어_프로그램 언어)-> 0001 0011 01001 (기계어)
* 컴퓨터(기계어) <-변환기- 고수준언어 (C 언어 C++ C# JAVA PYTHON GO) (고급언어에서 기계어로 컴파일러가 한다 하지만 다시 고급언어로 만드는것은 거의 불가능하다)
* 저수준는 언어 쉽게 변환 ex 어셉블리 (기계어를 다시 저수준 언어로 변경 쉽다)
* 변환기 = **컴파일러**
* 프로그램 실행 파일 .EXE 파일은 기계어로 이루어 짐
* 프로그램어 -> 코딩 -> 빌드(변환)-> 기계어
* 기계어를 다룰 수 있으면 파일을 보고 어떤 프로그램 인지 알 수 있고 변경이 가능하다 (어셈블리로 바꾸면 해석이 가능해진다)
* 언어 분류
  * 고급언어/저급언어(어셉블리)
* 고급언어 구분
  * 정적 언어 - 코딩->빌드->기계어 과정이 한번에 이루어 진다(컴파일언어) 속도가 빠르다
  * 동적 언어 - 코딩->빌드(있을 수도 있고 없을 수도 있다)(코딩,빌드 과정에서 기계어가 만들어지지 않는다)(프로그램이 동작할때 한줄씩 필요한부분만 컴파일어가 바꿔 실행) 속도가 느리다
    * 자바 중간상태가 존재 PYTHON, JAVASCR 중간상태 없다
* 각 언어마다 특징이 다르다 쓰임새가 다르다 여러 언어를 아는 것이 좋다
